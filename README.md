# Industrial Automation Digital Twin

# «

An educational industrial-automation prototype that combines an IEC 61131-3 Structured Text PLC program, a deterministic Python simulation model and a lightweight operations dashboard.

The simulated cell sorts products on a conveyor, counts good and rejected parts, supports automatic and manual modes, logs state changes and raises a preventive-maintenance alert after a configurable number of production cycles.

> \*\*Safety note\*\*: this repository is for education and simulation only. It must not be used to control real machinery without a qualified safety assessment, hardware safeguards, validated emergency circuits and compliance with applicable standards.

## Why this project

This project was built as an independent automation case study using **OpenPLC** as an open-source runtime target. It does **not** claim to develop or redistribute OpenPLC itself.

The project demonstrates:

* IEC 61131-3 Structured Text control logic;
* finite-state machine design;
* automatic/manual operating modes;
* emergency-stop behaviour and fault reset flow;
* part counting and reject handling;
* simulated event logging and operational metrics;
* automated Python tests for the reference logic;
* reproducible documentation and I/O mapping.

## Core features

|Area|Implemented behaviour|
|-|-|
|Operating states|Stopped, automatic, manual and emergency-stop states|
|Production|Conveyor control, part counting and quality decision inputs|
|Reject handling|Faulty parts are registered and a reject command is issued|
|Safety model|E-stop forces the system into a safe stopped state; manual reset is required|
|Maintenance|A configurable cycle threshold raises a preventive-maintenance flag|
|Observability|Event log, counters, state history and a dashboard|
|Validation|Unit tests validate the reference-state-machine behaviour|

## Quick start

### 1\. Run the simulation and tests

```bash
python -m simulator.demo\_run
python -m pytest -q
```

Expected result:

```text
5 passed
```

### 2\. Run the dashboard

```bash
pip install -r requirements.txt
streamlit run dashboard/app.py
```

### 3\. Run the PLC program

1. Install an OpenPLC runtime/editor compatible with IEC 61131-3 Structured Text.
2. Import [`plc/conveyor\_sorter.st`](plc/conveyor_sorter.st).
3. Use the addresses in [`docs/io\_mapping.md`](docs/io_mapping.md).
4. Test only in a simulated environment before connecting any hardware.

## Engineering assumptions

* The quality decision is sampled together with the part-detection edge.
* The reject cylinder is represented as a timed command in the PLC program and a logged action in the simulator.
* The E-stop model here is a **software simulation**, not a replacement for a hardwired safety circuit.
* The maintenance threshold is configurable and represents a simple preventive-maintenance policy.

## Results from the included demo scenario

The deterministic demo scenario produces:

* 4 processed parts;
* 3 accepted parts;
* 1 rejected part;
* one manual-mode transition;
* one emergency-stop event and controlled reset.

The resulting event log is stored in [`data/demo\_events.jsonl`](data/demo_events.jsonl).
